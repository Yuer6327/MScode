# Microsoft VS Code 塔防游戏设计文档

## 游戏概念
"Microsoft VS Code"是一个塔防策略游戏，灵感来自植物大战僵尸。玩家扮演VS Code开发者，需要部署各种开发工具、扩展和调试器来抵御不断入侵的代码错误、漏洞和恶意软件。

## 游戏主题和视觉风格
- **主题**: 现代开发环境美学，结合VS Code的界面设计
- **色彩方案**: 深色主题为主，使用VS Code经典的蓝色、橙色、紫色点缀
- **视觉语言**: 简洁现代的UI设计，符合开发者工具的美学
- **字体**: 使用等宽字体和现代化的无衬线字体

## 游戏机制

### 游戏网格
- 5行9列的网格布局
- 每行代表一个代码文件或项目路径
- 敌人从右侧入侵，向左移动试图"破坏"代码

### 防御单位（VS Code工具）
1. **基础调试器** - 发射调试光束攻击单个错误
2. **代码格式化器** - 范围攻击，清理多个格式错误
3. **Git版本控制** - 可以撤销敌人的攻击（恢复生命值）
4. **IntelliSense智能感知** - 减速敌人移动速度
5. **终端命令行** - 高伤害但攻击速度慢
6. **扩展插件** - 提供各种增益效果
7. **代码审查** - 揭示隐形敌人
8. **自动保存** - 被动防御，减少受到的伤害

### 敌人类型（代码错误）
1. **语法错误** - 基础敌人，血量低
2. **运行时错误** - 中等血量，速度较快
3. **逻辑漏洞** - 高血量，需要多次攻击
4. **安全漏洞** - 会跳过某些防御
5. **依赖冲突** - 范围伤害，影响周围单位
6. **内存泄漏** - 每回合回复生命值
7. **编译错误** - 高防御，需要特殊工具对付
8. **僵尸进程** - 死亡后分裂成小错误

### 资源系统
- **代码质量点数**: 主要资源，用于部署防御单位
- **CPU使用率**: 限制同时部署的单位数量
- **内存占用**: 特殊技能的能量来源

### 游戏进程
- **关卡系统**: 50个关卡，每10关有一个Boss战
- **难度递增**: 敌人血量、速度、数量逐步增加
- **新单位解锁**: 通过完成关卡获得新的VS Code工具

## 特殊机制

### Boss关卡
- **第10关**: "无限循环" - 生成大量小错误
- **第20关**: "堆栈溢出" - 高血量Boss，会召唤小弟
- **第30关**: "依赖地狱" - 会禁用玩家的某些工具
- **第40关**: "编译器崩溃" - 随机改变游戏规则
- **第50关**: "生产环境" - 最终Boss，结合所有机制

### 天气效果
- **代码风暴**: 某些行的敌人移动速度加快
- **调试模式**: 玩家可以看到敌人的血量条
- **夜间模式**: 某些单位获得增强效果

### 成就系统
- **完美主义者**: 无伤通关
- **效率专家**: 快速通关
- **工具大师**: 使用所有类型的单位
- **调试高手**: 击败特定数量的错误

## 用户界面
- **主菜单**: 开始游戏、设置、成就、帮助
- **游戏界面**: 网格、工具栏、资源显示、暂停按钮
- **升级界面**: 单位升级、技能树
- **商店界面**: 购买新单位、皮肤

## 技术实现
- **游戏引擎**: HTML5 Canvas + JavaScript
- **动画库**: Anime.js
- **视觉效果**: PIXI.js
- **音效**: 使用代码主题音效（键盘敲击、编译声音等）
